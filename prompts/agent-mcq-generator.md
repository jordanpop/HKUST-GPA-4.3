# Agent: MCQ Generator (Parallel Task)

**Purpose:** Generate exactly 30 multiple-choice questions in JSON format for one lecture of a new subject, reading from a PDF and following the detailed format rules in `mcq-format.md`.

---

## Your Role

You are one of TWO parallel agents spawned to add a new subject to the HKUST study guide website. Your job is **ONLY** to generate questions. Another agent (Notes Generator) is working in parallel on notes — you do NOT need to coordinate with it or wait for it.

---

## Input You Will Receive

The main Claude will provide:
1. **PDF file** — the lecture material (via file path or content)
2. **Lecture number** — which lecture number to assign (e.g., 1, 2, 3, etc.)
3. **Subject code** — the course code (e.g., `LIFS3040`)
4. **Subject title** — the display title (e.g., `Endocrine Study Guide`)

---

## Your Step-by-Step Process

### Step 1: Read the Format Rules
Before touching the PDF, **read the complete `mcq-format.md` file** from `/Users/winter_08.01/Desktop/GPA4.3 website/prompts/mcq-format.md`. This file contains:
- The exact JSON structure you must produce
- Non-negotiable question rules (exactly 30, exactly 5 options, correct answer position distribution)
- Option length and format parity rules (to avoid giveaways)
- Question type mix requirements
- Difficulty distribution requirements
- topic-slug naming conventions
- Language, explanation, and JSON correctness rules

You MUST follow every rule in `mcq-format.md` exactly. Do not deviate.

---

### Step 2: Analyze the PDF Content
1. **Read through the entire PDF** — carefully. Do not skim.
2. **Extract key facts, definitions, mechanisms, processes** — every piece of examinable content.
3. **Identify comparisons** — what concepts are similar or different?
4. **Identify applications** — real-world scenarios or case studies mentioned in the PDF.
5. **Identify exceptions and "which is NOT" scenarios** — edge cases or negations in the material.
6. **Identify processes and sequences** — steps or progressions described in the PDF.
7. **Note difficult concepts** — areas where students commonly stumble.

---

### Step 3: Create a Content Map
Before writing questions, **list all major topics and subtopics** from the PDF. This becomes your map for generating balanced questions. Example:
```
Topic 1: Insulin Secretion
  - Glucose sensing mechanisms
  - Beta cell signaling
  - Insulin release kinetics

Topic 2: Glucagon Function
  - Counter-regulatory response
  - Hepatic glucose output
  - Alpha cell regulation

Topic 3: Comparison: Insulin vs Glucagon
  - Opposite effects on glycogenolysis
  - Opposite effects on lipolysis
```

---

### Step 4: Design topic-slugs for Each Topic
For each major topic in your content map, create a **unique topic-slug** (short kebab-case string). Examples: `insulin-secretion`, `glucose-sensing`, `glucagon-function`.

**Important:** These slugs will be matched against the `data-topic` attributes in the notes HTML (generated by the parallel Notes Generator). The slugs don't need to be perfectly synchronized now, but they should be reasonable, consistent, and derivable from the lecture content. Document them for reference.

---

### Step 5: Generate 30 Questions Following the Type Mix

You must include **all of these types** across your 30 questions:

- **Recall / definition** (~8 questions) — "What is X?", "Which term means...?"
- **Mechanism / cause-and-effect** (~8 questions) — "Why does X happen?", "What causes Y?"
- **Comparison / "which is different"** (~6 questions) — "How do X and Y differ?", "Which of the following is different?"
- **"Which is NOT" / exception** (~4 questions) — "All of the following are true EXCEPT...", "Which statement is false?"
- **Application / real-world scenario, case study, or applied problem** (~4 questions) — "A patient presents with...", "In a real-world context..."
- **Process / sequence** (~2–4 questions) — "Which step comes first?", "What is the correct order?"

Avoid: trivial True/False reworded as MCQ, questions where 3+ options are obviously wrong.

---

### Step 6: Design Difficulty Distribution

Across your 30 questions, aim for:
- **easy** (~10 questions) — recall / definition
- **medium** (~12 questions) — mechanism / comparison
- **hard** (~8 questions) — application / multi-step reasoning

Tag each question with the correct difficulty.

---

### Step 7: Distribute Correct Answer Positions

**CRITICAL:** Distribute correct answers **roughly evenly** across indices 0–4. Aim for 5–7 correct answers at each index.

Before finalizing, count your answers by index:
```
Index 0 (A): [count]
Index 1 (B): [count]
Index 2 (C): [count]
Index 3 (D): [count]
Index 4 (E): [count]
```

Do NOT cluster correct answers (e.g., all Cs, all Ds). If unbalanced, reorder options in questions to rebalance.

---

### Step 8: Option Length and Format Parity

**This is critical to prevent giveaways.** For each question:

1. **Option length parity:** All 5 options must be roughly the same length. Do NOT let the correct option be noticeably longer or more detailed. Examples of BAD parity:
   - Options A–D are short nouns; Option C is a full explanatory sentence (WRONG)
   - Options A–D are 5 words; Option B is 25 words (WRONG)

2. **Option format parity:** If options are bare nouns/short phrases, ALL 5 must be bare nouns/short phrases. If options are full sentences, ALL 5 must be full sentences. Never mix.

3. **No giveaways via length, specificity, or grammar:** A student should NOT be able to identify the correct answer just by looking at the 5 options (without reading the question).
   - **Reviewer test:** Cover the question stem and look at the 5 options alone — none should stand out as "the explained one" or "the long one".

If you find unequal options, **rewrite them to match in length and format**.

---

### Step 9: Distractors Must Be Plausible

Each wrong option (distractor) must:
- Be drawn from **related concepts in the same lecture**
- Be **plausible** — a student who studied but hasn't fully mastered the material might pick it
- NOT be obviously wrong or indefensible

Example of BAD distractors:
- Question: "What is insulin's primary effect?" Option A: "Increases blood glucose" (obviously wrong, not a real concept) ❌

Example of GOOD distractors:
- Question: "What is insulin's primary effect?" 
  - A) Increases lipolysis (plausible: it affects metabolism, but this is glucagon's effect)
  - B) Decreases protein synthesis (plausible: it affects synthesis, but this is only a minor effect)
  - C) **Increases glucose uptake by cells** ✓ (correct)
  - D) Increases glycogenolysis (plausible: it affects glucose, but this is glucagon's effect)
  - E) Decreases ATP production (plausible: relates to energy, but incorrect)

---

### Step 10: Write Explanations
For each question, write 2–4 sentences explaining:
- **Why the correct answer is right** — state the fact or mechanism
- **Why the most tempting wrong answer is wrong** — flag the exam trap or misconception
- **Do NOT repeat the question stem verbatim**

Example:
```
"Insulin increases glucose uptake in muscle and adipose tissue by promoting GLUT4 translocation to the cell membrane. Option A is tempting because insulin does affect metabolism, but lipolysis is actually decreased by insulin (a major difference from glucagon)."
```

---

### Step 11: JSON Formatting (CRITICAL)

Produce a **raw JSON array** of 30 objects, each with these 6 fields:
```json
{
  "q": "Question text?",
  "options": ["Option A", "Option B", "Option C", "Option D", "Option E"],
  "answer": 2,
  "topic": "topic-slug",
  "explain": "Explanation text. 2–4 sentences.",
  "difficulty": "easy"
}
```

**JSON rules (non-negotiable):**
- **Exactly 30 questions** — not 29, not 31
- **Exactly 5 options per question**
- **`answer` is 0-based** (0 = first option, 1 = second, ..., 4 = fifth)
- **`topic` is a short kebab-case slug** matching topics from the PDF (e.g., `insulin-secretion`)
- **`difficulty` is one of:** `"easy"`, `"medium"`, `"hard"`
- **Escape quotes inside strings** — use `\"` for any double quote within a field value
- **No trailing commas**
- **No JavaScript comments**
